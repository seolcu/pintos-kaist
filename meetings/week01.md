핀토스 프로젝트 개요 및 목표
- 핀토스는 MIT와 스탠포드에서 사용된 오래된 운영체제 학습 도구
- AI 문제 해결로 인해 현재는 사용이 어렵지만, 시스템 프로그래밍 학습에 유용
- 프로젝트 목표는 컴퓨터 시스템 내부 이해도 증진 및 프로그래밍 실력 향상

핀토스 프로젝트 진행 방식 및 환경 설정
- 프로젝트는 과제 형태로 진행되며, 가상머신에서 실제 코드 실행
- 리눅스와 유사한 교육 환경 제공, 직접 리눅스 구현보다 효율적
- QEMU 및 스크립트를 활용한 환경 설정 및 테스트 케이스 확인

첫 번째 과제: 스케줄러 구현
- 첫 번째 과제는 스케줄러 구현으로, 스레드 관장 및 실행 구조 생성
- 운영체제 커널 모드에서 동작하는 스레드 스케줄링 구현
- 로더 및 이미지를 통해 프로그램 로딩 방식과 커널 초기화 이해

두 번째 과제: 동기화 및 우선순위 스케줄링
- 두 번째 과제는 동기화 구현, 특히 인터럽트 제어를 통한 임계 영역 보호
- 타이머 인터럽트를 활용한 슬립 및 웨이크업 기능 구현
- 우선순위 역전 해결을 위한 우선순위 상속 프로토콜 구현

세 번째 과제: 가상 메모리 관리
- 세 번째 과제는 가상 메모리 관리로, 난이도가 높음
- 익명 페이지, 지연 로딩, Copy-on-Write, 스왑 영역 구현 포함
- 파일 시스템과 메모리 매핑을 통한 페이지 처리 구현

네 번째 과제: 파일 시스템 구현
- 네 번째 과제는 파일 시스템 구현으로, 상대적으로 쉬운 난이도
- 기존 파일 시스템을 수정하여 새로운 파일 시스템 생성
- 프로젝트 전체 일정은 4주로 예상되며, 3번 과제까지 완료 목표
